# Reference: File Templates

Complete templates for each file generated by `setup-vercel`.

Placeholders use `{{VARIABLE}}` syntax — replace with collected user values.

## `infra/vercel/{{APP_NAME}}/src/variables.tf`

```hcl
variable "ENVIRONMENT" {
  type        = string
  description = "Target environment (development or production)"
}

variable "environment_variables" {
  type        = map(string)
  default     = {}
  description = "Environment variables to set on the Vercel project"
}

variable "vercel" {
  type = object({
    project = object({
      name                       = string
      root_directory             = optional(string)
      build_command              = optional(string)
      framework                  = optional(string)
      public_source              = optional(bool, false)
      output_directory           = optional(string)
      install_command             = optional(string)
      auto_assign_custom_domains = optional(bool, true)
      vercel_authentication = optional(object({
        deployment_type = string
      }), { deployment_type = "none" })
    })
    project_domains = optional(list(object({
      domain   = string
      redirect = optional(string)
    })), [])
  })
  description = "Vercel project configuration"
}

variable "VERCEL_TOKEN" {
  type        = string
  sensitive   = true
  description = "Vercel API token"
}

variable "VERCEL_ORG_ID" {
  type        = string
  default     = ""
  description = "Vercel team/org ID (leave empty for personal account)"
}
```

## `infra/vercel/{{APP_NAME}}/src/main.tf`

```hcl
locals {
  environments = {
    "development" = "dev"
    "production"  = "prod"
  }
  environment_short_code = lookup(local.environments, lower(var.ENVIRONMENT), "")
}

resource "vercel_project" "main" {
  team_id                    = var.VERCEL_ORG_ID != "" ? var.VERCEL_ORG_ID : null
  name                       = "${var.vercel.project.name}-${local.environment_short_code}"
  output_directory           = var.vercel.project.output_directory
  build_command              = var.vercel.project.build_command
  root_directory             = var.vercel.project.root_directory
  framework                  = var.vercel.project.framework
  public_source              = var.vercel.project.public_source
  install_command             = var.vercel.project.install_command
  auto_assign_custom_domains = var.vercel.project.auto_assign_custom_domains

  dynamic "vercel_authentication" {
    for_each = var.vercel.project.vercel_authentication != null ? [var.vercel.project.vercel_authentication] : []
    content {
      deployment_type = vercel_authentication.value.deployment_type
    }
  }
}

resource "vercel_project_domain" "primary" {
  for_each = {
    for d in var.vercel.project_domains : d.domain => d
    if d.redirect == null || d.redirect == ""
  }

  project_id = vercel_project.main.id
  team_id    = var.VERCEL_ORG_ID != "" ? var.VERCEL_ORG_ID : null
  domain     = each.value.domain
}

resource "vercel_project_domain" "redirect" {
  for_each = {
    for d in var.vercel.project_domains : d.domain => d
    if d.redirect != null && d.redirect != ""
  }

  project_id          = vercel_project.main.id
  team_id             = var.VERCEL_ORG_ID != "" ? var.VERCEL_ORG_ID : null
  domain              = each.value.domain
  redirect            = each.value.redirect
  redirect_status_code = 308
}

resource "vercel_project_environment_variable" "variables" {
  for_each = var.environment_variables

  project_id = vercel_project.main.id
  team_id    = var.VERCEL_ORG_ID != "" ? var.VERCEL_ORG_ID : null
  key        = each.key
  value      = each.value
  target     = ["production", "preview", "development"]
}

output "VERCEL_PROJECT_ID" {
  description = "The Vercel project ID"
  value       = vercel_project.main.id
}
```

## `infra/vercel/{{APP_NAME}}/src/config.development.json`

Template — fill with user values:

```json
{
  "environment_variables": {},
  "vercel": {
    "project_domains": [],
    "project": {
      "name": "{{PROJECT_NAME}}",
      "root_directory": "{{ROOT_DIRECTORY}}",
      "build_command": "{{BUILD_COMMAND}}",
      "output_directory": "{{OUTPUT_DIRECTORY}}",
      "framework": {{FRAMEWORK}},
      "public_source": false,
      "auto_assign_custom_domains": true,
      "vercel_authentication": {
        "deployment_type": "none"
      }
    }
  }
}
```

Notes:
- `{{FRAMEWORK}}` should be `null` (no quotes) or `"nextjs"`, `"astro"`, etc.
- Development config typically has no custom domains.

```json
    "project": {
      "name": "{{PROJECT_NAME}}",
      "root_directory": "{{ROOT_DIRECTORY}}",
      "build_command": "{{BUILD_COMMAND}}",
```

- Remove `project_domains` array entries if no custom domain is provided.
- Add additional env vars to `environment_variables` as needed.

## `.github/workflows/{{APP_NAME}}-vercel-deploy.yml`

```yaml
name: {{APP_DISPLAY_NAME}} [Vercel] Deploy

on:
  push:
    branches: [main]
  workflow_dispatch:
    inputs:
      environment:
        description: "Target environment"
        required: true
        type: choice
        options:
          - development
          - production
        default: development

permissions:
  contents: write
  actions: write

jobs:
  check-changes:
    runs-on: ubuntu-latest
    name: Check for changes
    outputs:
      infra-changed: ${{ steps.check.outputs.infra-changed }}
      app-changed: ${{ steps.check.outputs.app-changed }}
      environment: ${{ steps.resolve-env.outputs.environment }}

    steps:
      - name: Checkout
        uses: actions/checkout@v4
        with:
          fetch-depth: 2

      - name: Setup pnpm
        uses: pnpm/action-setup@v5

      - name: Setup Node.js
        uses: actions/setup-node@v6
        with:
          node-version: "{{NODE_VERSION}}"
          cache: "pnpm"

      - name: Install dependencies
        run: pnpm install

      - name: Run code quality checks
        run: pnpm check

      - name: Resolve environment
        id: resolve-env
        run: |
          if [ "${{ github.event_name }}" = "workflow_dispatch" ]; then
            echo "environment=${{ github.event.inputs.environment }}" >> $GITHUB_OUTPUT
          else
            echo "environment=development" >> $GITHUB_OUTPUT
          fi

      - name: Check for changes
        id: check
        run: |
          if [ "${{ github.event_name }}" = "workflow_dispatch" ]; then
            echo "infra-changed=true" >> $GITHUB_OUTPUT
            echo "app-changed=true" >> $GITHUB_OUTPUT
            exit 0
          fi
          if git diff --name-only HEAD~1 -- infra/vercel/{{APP_NAME}}/ | grep -q .; then
            echo "infra-changed=true" >> $GITHUB_OUTPUT
          else
            echo "infra-changed=false" >> $GITHUB_OUTPUT
          fi
          if git diff --name-only HEAD~1 -- {{APP_PATH}}/ package.json pnpm-lock.yaml | grep -q .; then
            echo "app-changed=true" >> $GITHUB_OUTPUT
          else
            echo "app-changed=false" >> $GITHUB_OUTPUT
          fi

  infra-deploy:
    needs: check-changes
    if: needs.check-changes.outputs.infra-changed == 'true'
    runs-on: ubuntu-latest
    name: Infrastructure Deploy
    outputs:
      project-id: ${{ steps.export-outputs.outputs.project-id }}
    env:
      TF_VAR_ENVIRONMENT: ${{ needs.check-changes.outputs.environment }}
      DOTENV_PRIVATE_KEY: ${{ secrets.DOTENV_PRIVATE_KEY }}

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup pnpm
        uses: pnpm/action-setup@v5

      - name: Setup Node.js
        uses: actions/setup-node@v6
        with:
          node-version: "{{NODE_VERSION}}"
          cache: "pnpm"

      - name: Load secrets from encrypted .env
        run: |
          pnpm dlx @dotenvx/dotenvx run --env-file=infra/vercel/{{APP_NAME}}/.env.${{ env.TF_VAR_ENVIRONMENT }} -- sh -c '
            echo "TF_VAR_VERCEL_TOKEN=$VERCEL_TOKEN" >> "$GITHUB_ENV"
            echo "TF_VAR_VERCEL_ORG_ID=$VERCEL_ORG_ID" >> "$GITHUB_ENV"
            echo "TERRAFORM_STATE_ENCRYPT_KEY=$TERRAFORM_STATE_ENCRYPT_KEY" >> "$GITHUB_ENV"
          '

      - name: Fetch Terraform State from state branch
        working-directory: infra/vercel/{{APP_NAME}}/src
        run: |
          STATE_FILE="terraform-${{ env.TF_VAR_ENVIRONMENT }}.tfstate.enc"
          if git fetch origin terraform-state 2>/dev/null; then
            if git show "origin/terraform-state:infra/vercel/{{APP_NAME}}/src/${STATE_FILE}" > "$STATE_FILE" 2>/dev/null && [ -s "$STATE_FILE" ]; then
              echo "State file fetched"
            else
              rm -f "$STATE_FILE"
            fi
          fi

      - name: Decrypt Terraform State
        working-directory: infra/vercel/{{APP_NAME}}/src
        run: |
          STATE_FILE="terraform-${{ env.TF_VAR_ENVIRONMENT }}.tfstate.enc"
          if [ -s "$STATE_FILE" ]; then
            if echo "$TERRAFORM_STATE_ENCRYPT_KEY" | openssl enc -aes-256-cbc -d -in "$STATE_FILE" -out "terraform.tfstate" -pass stdin 2>/dev/null; then
              echo "State file decrypted"
            else
              echo "::warning::Failed to decrypt state — proceeding without existing state"
              rm -f terraform.tfstate
            fi
          fi

      - name: Setup Terraform
        uses: hashicorp/setup-terraform@v4
        with:
          terraform_version: "{{TERRAFORM_VERSION}}"
          terraform_wrapper: false

      - name: Terraform Init
        working-directory: infra/vercel/{{APP_NAME}}/src
        run: terraform init

      - name: Import existing project if state is missing
        working-directory: infra/vercel/{{APP_NAME}}/src
        continue-on-error: true
        run: |
          if [ ! -f "terraform.tfstate" ] || [ "$(jq '.resources | length' terraform.tfstate 2>/dev/null)" = "0" ]; then
            ENVIRONMENT="${{ needs.check-changes.outputs.environment }}"
            case "$ENVIRONMENT" in
              development) ENV_SHORT="dev" ;;
              production)  ENV_SHORT="prod" ;;
              *)           ENV_SHORT="${ENVIRONMENT:0:3}" ;;
            esac
            PROJECT_NAME="{{PROJECT_NAME}}-${ENV_SHORT}"
            TEAM_PARAM=""
            if [ -n "${TF_VAR_VERCEL_ORG_ID}" ]; then
              TEAM_PARAM="?teamId=${TF_VAR_VERCEL_ORG_ID}"
            fi
            PROJECT_ID=$(curl -sf -H "Authorization: Bearer ${TF_VAR_VERCEL_TOKEN}" "https://api.vercel.com/v9/projects/${PROJECT_NAME}${TEAM_PARAM}" | jq -r '.id // empty')
            if [ -n "$PROJECT_ID" ]; then
              terraform import -var-file="config.${ENVIRONMENT}.json" vercel_project.main "$PROJECT_ID"

              # Import existing domain resources
              DOMAINS=$(jq -r '.vercel.project_domains[]?.domain // empty' "config.${ENVIRONMENT}.json" 2>/dev/null)
              for DOMAIN in $DOMAINS; do
                REDIRECT=$(jq -r --arg d "$DOMAIN" '.vercel.project_domains[] | select(.domain == $d) | .redirect // empty' "config.${ENVIRONMENT}.json" 2>/dev/null)
                if [ -z "$REDIRECT" ]; then
                  terraform import -var-file="config.${ENVIRONMENT}.json" "vercel_project_domain.primary[\"${DOMAIN}\"]" "${PROJECT_ID}/${DOMAIN}" || true
                else
                  terraform import -var-file="config.${ENVIRONMENT}.json" "vercel_project_domain.redirect[\"${DOMAIN}\"]" "${PROJECT_ID}/${DOMAIN}" || true
                fi
              done
            fi
          fi

      - name: Terraform Plan
        working-directory: infra/vercel/{{APP_NAME}}/src
        run: terraform plan -var-file="config.${{ env.TF_VAR_ENVIRONMENT }}.json" -out=tfplan

      - name: Terraform Apply
        working-directory: infra/vercel/{{APP_NAME}}/src
        run: terraform apply -auto-approve tfplan

      - name: Export Terraform Outputs
        id: export-outputs
        working-directory: infra/vercel/{{APP_NAME}}/src
        run: |
          PROJECT_ID=$(terraform output -raw VERCEL_PROJECT_ID)
          echo "project-id=$PROJECT_ID" >> $GITHUB_OUTPUT

      - name: Encrypt Terraform State
        working-directory: infra/vercel/{{APP_NAME}}/src
        run: |
          if [ -f "terraform.tfstate" ]; then
            echo "$TERRAFORM_STATE_ENCRYPT_KEY" | openssl enc -aes-256-cbc -in "terraform.tfstate" -out "terraform-${{ env.TF_VAR_ENVIRONMENT }}.tfstate.enc" -pass stdin
            rm -f terraform.tfstate
          else
            echo "::error::No terraform.tfstate found"
            exit 1
          fi

      - name: Configure Git
        run: |
          git config --global user.name "github-actions[bot]"
          git config --global user.email "github-actions[bot]@users.noreply.github.com"

      - name: Commit Encrypted State to state branch
        working-directory: infra/vercel/{{APP_NAME}}/src
        run: |
          STATE_FILE="terraform-${{ env.TF_VAR_ENVIRONMENT }}.tfstate.enc"
          cp "$STATE_FILE" "/tmp/$STATE_FILE"
          cd ${{ github.workspace }}
          if git fetch origin terraform-state 2>/dev/null; then
            git checkout terraform-state
          else
            git checkout --orphan terraform-state
            git rm -rf . 2>/dev/null || true
          fi
          mkdir -p infra/vercel/{{APP_NAME}}/src
          cp "/tmp/$STATE_FILE" "infra/vercel/{{APP_NAME}}/src/$STATE_FILE"
          git add "infra/vercel/{{APP_NAME}}/src/$STATE_FILE"
          git diff --staged --quiet && echo "No state changes" && exit 0
          git commit -m "chore: update terraform state for ${{ env.TF_VAR_ENVIRONMENT }}"
          git push origin terraform-state

  deploy-preview:
    needs: [check-changes, infra-deploy]
    if: |
      always() &&
      needs.infra-deploy.result != 'failure' &&
      (needs.check-changes.outputs.app-changed == 'true' || needs.check-changes.outputs.infra-changed == 'true')
    runs-on: ubuntu-latest
    name: Deploy Preview
    outputs:
      preview-url: ${{ steps.preview-deploy.outputs.preview-url }}
    env:
      DOTENV_PRIVATE_KEY: ${{ secrets.DOTENV_PRIVATE_KEY }}

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup pnpm
        uses: pnpm/action-setup@v5

      - name: Setup Node.js
        uses: actions/setup-node@v6
        with:
          node-version: "{{NODE_VERSION}}"
          cache: "pnpm"

      - name: Load secrets from encrypted .env
        run: |
          pnpm dlx @dotenvx/dotenvx run --env-file=infra/vercel/{{APP_NAME}}/.env.${{ needs.check-changes.outputs.environment }} -- sh -c '
            echo "VERCEL_TOKEN=$VERCEL_TOKEN" >> "$GITHUB_ENV"
            echo "VERCEL_ORG_ID=$VERCEL_ORG_ID" >> "$GITHUB_ENV"
          '

      - name: Resolve Vercel Project ID
        run: |
          PROJECT_ID="${{ needs.infra-deploy.outputs.project-id }}"
          if [ -z "$PROJECT_ID" ]; then
            ENVIRONMENT="${{ needs.check-changes.outputs.environment }}"
            case "$ENVIRONMENT" in
              production) ENV_SHORT="prod" ;;
              *)          ENV_SHORT="dev" ;;
            esac
            PROJECT_NAME="{{PROJECT_NAME}}-${ENV_SHORT}"
            TEAM_PARAM=""
            if [ -n "${VERCEL_ORG_ID}" ]; then TEAM_PARAM="?teamId=${VERCEL_ORG_ID}"; fi
            PROJECT_ID=$(curl -sf -H "Authorization: Bearer ${VERCEL_TOKEN}" "https://api.vercel.com/v9/projects/${PROJECT_NAME}${TEAM_PARAM}" | jq -r '.id // empty')
          fi
          echo "VERCEL_PROJECT_ID=$PROJECT_ID" >> $GITHUB_ENV

      - name: Install dependencies
        run: pnpm install

      - name: Install Vercel CLI
        run: npm install -g vercel@latest

      - name: Pull Vercel Preview Environment
        run: vercel pull --yes --environment=preview --token=$VERCEL_TOKEN

      - name: Build Preview Artifacts
        run: vercel build --token=$VERCEL_TOKEN

      - name: Deploy to Vercel Preview
        id: preview-deploy
        run: |
          PREVIEW_URL=$(vercel deploy --prebuilt --token=$VERCEL_TOKEN)
          echo "preview-url=$PREVIEW_URL" >> $GITHUB_OUTPUT

  e2e-tests:
    needs: [deploy-preview]
    if: always() && needs.deploy-preview.result == 'success'
    runs-on: ubuntu-latest
    name: E2E Tests

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup pnpm
        uses: pnpm/action-setup@v5

      - name: Setup Node.js
        uses: actions/setup-node@v6
        with:
          node-version: "{{NODE_VERSION}}"
          cache: "pnpm"

      - name: Install dependencies
        run: pnpm install

      - name: Install Playwright Browsers
        run: {{E2E_PLAYWRIGHT_INSTALL_COMMAND}}

      - name: Wait for Preview URL readiness
        run: |
          PREVIEW_URL="${{ needs.deploy-preview.outputs.preview-url }}"
          echo "Waiting for preview deployment: ${PREVIEW_URL}"

          for attempt in {1..30}; do
            HTTP_STATUS=$(curl -s -L -o /dev/null -w "%{http_code}" "${PREVIEW_URL}")
            if [ "${HTTP_STATUS}" -ge 200 ] && [ "${HTTP_STATUS}" -lt 400 ]; then
              echo "Preview is ready (HTTP ${HTTP_STATUS})"
              exit 0
            fi

            echo "Attempt ${attempt}/30: preview not ready yet (HTTP ${HTTP_STATUS})"
            sleep 10
          done

          echo "::error::Preview deployment did not become ready in time"
          exit 1

      - name: Run E2E Tests
        run: {{E2E_TEST_COMMAND}}
        env:
          PLAYWRIGHT_BASE_URL: ${{ needs.deploy-preview.outputs.preview-url }}
          CI: true

      - name: Upload visual test artifacts
        if: failure()
        uses: actions/upload-artifact@v4
        with:
          name: visual-test-results
          path: |
            {{APP_PATH}}/test-results/
            {{APP_PATH}}/playwright-report/
          retention-days: 7

  deploy-production:
    needs: [check-changes, infra-deploy, deploy-preview, e2e-tests]
    if: |
      always() &&
      needs.deploy-preview.result == 'success' &&
      needs.e2e-tests.result == 'success' &&
      needs.infra-deploy.result != 'failure'
    runs-on: ubuntu-latest
    name: Deploy Production
    env:
      DOTENV_PRIVATE_KEY: ${{ secrets.DOTENV_PRIVATE_KEY }}

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup pnpm
        uses: pnpm/action-setup@v5

      - name: Setup Node.js
        uses: actions/setup-node@v6
        with:
          node-version: "{{NODE_VERSION}}"
          cache: "pnpm"

      - name: Load secrets from encrypted .env
        run: |
          pnpm dlx @dotenvx/dotenvx run --env-file=infra/vercel/{{APP_NAME}}/.env.${{ needs.check-changes.outputs.environment }} -- sh -c '
            echo "VERCEL_TOKEN=$VERCEL_TOKEN" >> "$GITHUB_ENV"
            echo "VERCEL_ORG_ID=$VERCEL_ORG_ID" >> "$GITHUB_ENV"
          '

      - name: Resolve Vercel Project ID
        run: |
          PROJECT_ID="${{ needs.infra-deploy.outputs.project-id }}"
          if [ -z "$PROJECT_ID" ]; then
            ENVIRONMENT="${{ needs.check-changes.outputs.environment }}"
            case "$ENVIRONMENT" in

              *)          ENV_SHORT="dev" ;;
            esac
            PROJECT_NAME="{{PROJECT_NAME}}-${ENV_SHORT}"
            TEAM_PARAM=""
            if [ -n "${VERCEL_ORG_ID}" ]; then TEAM_PARAM="?teamId=${VERCEL_ORG_ID}"; fi
            PROJECT_ID=$(curl -sf -H "Authorization: Bearer ${VERCEL_TOKEN}" "https://api.vercel.com/v9/projects/${PROJECT_NAME}${TEAM_PARAM}" | jq -r '.id // empty')

      - name: Install dependencies
        run: pnpm install

      - name: Install Vercel CLI
        run: npm install -g vercel@latest

      - name: Pull Vercel Production Environment
        run: vercel pull --yes --environment=production --token=$VERCEL_TOKEN

      - name: Build Production Artifacts
```

## Placeholder Reference

| Placeholder | Description | Example |
|-------------|-------------|---------|
| `{{APP_NAME}}` | Directory slug for the app | `web` |
| `{{APP_DISPLAY_NAME}}` | Human-readable name for workflow | `Web` |
| `{{APP_PATH}}` | Monorepo path to the app source | `apps/my-app` |
| `{{PROJECT_NAME}}` | Vercel project base name (suffixed with -dev/-prod) | `my-app-web` |
| `{{ROOT_DIRECTORY}}` | Vercel root directory setting | `apps/my-app` |
| `{{BUILD_COMMAND}}` | Build command | `npm run build` |
| `{{OUTPUT_DIRECTORY}}` | Build output dir | `dist` or `.next` |
| `{{FRAMEWORK}}` | Vercel framework preset (`null` or string) | `null` / `"nextjs"` |
| `{{PRODUCTION_DOMAIN}}` | Primary production domain | `example.com` |
| `{{NODE_VERSION}}` | Node.js version | `24` |
| `{{TERRAFORM_VERSION}}` | Terraform version | `1.10.5` |
| `{{E2E_TEST_COMMAND}}` | E2E test run command | `pnpm --filter my-app run test:e2e` |
| `{{E2E_PLAYWRIGHT_INSTALL_COMMAND}}` | Playwright browser install | `pnpm --filter my-app exec playwright install --with-deps chromium` |
| `{{VERCEL_TOKEN_PLACEHOLDER}}` | Placeholder for Vercel token | `<your-vercel-token>` |
| `{{VERCEL_ORG_ID_PLACEHOLDER}}` | Placeholder for Vercel org ID | `<your-vercel-org-id>` |
| `{{STATE_ENCRYPT_KEY_PLACEHOLDER}}` | Placeholder for state encryption key | `<run: openssl rand -base64 32>` |
